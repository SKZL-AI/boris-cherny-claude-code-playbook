Boris' PRs in 30 Tagen

Boris Cherny ist der Erfinder und Head of Claude Code bei Anthropic. Er startete das Projekt im September 2024 als Side-Project; Launch war am 24. Februar 2025. Seine wichtigste einzelne Regel: "Give Claude a way to verify its work — it will 2–3× the quality."

Sein Setup ist überraschend simpel: 5 parallele Terminal-Tabs + 5–10 Web-Sessions auf claude.ai/code + mobile Sessions, immer zuerst im Plan-Mode, mit einer geteilten CLAUDE.md, Slash-Commands in .claude/commands/, Subagents in .claude/agents/, einem PostToolUse-Format-Hook und /permissions (nie --dangerously-skip-permissions außerhalb von Sandboxes).

Der Bogen seiner Tipps von Launch bis heute: "Chatbot im Terminal" → "Paralleler Execution-Engine" → "Autonomer, geplanter, sprachgesteuerter Agent". Plan-Mode und CLAUDE.md (Q1 2026) → Hooks/Plugins (Feb 2026) → /simplify, /batch, Worktrees (Feb–Mär 2026) → /loop, /schedule, Code Review, /btw (Mär 2026) → Auto Mode, /focus, /go für Opus 4.7 (Apr 2026).

Staged Adoption — Boris' eigene Empfehlung

Vom Vanilla-Setup zum autonomen Agent-Stack. Klare Schwellen, ab wann du in die nächste Phase gehst.

1
Vanilla
Woche 1
Claude Code installieren (CLI oder VS Code Extension). /init für Starter-CLAUDE.md. Plan-Mode (Shift+Tab zweimal) für jeden Task. Nicht customizen.
Graduate when: 5+ PRs geshippt, Claude macht den gleichen Fehler zweimal — Zeit für Regeln in CLAUDE.md.
2
Compounding Knowledge
Woche 2–3
CLAUDE.md wie Code behandeln: gnadenlos prunen, Regel hinzufügen bei jedem Fehler. 1–2 Slash-Commands für tägliche Workflows hinzufügen.
Graduate when: CLAUDE.md > 1 Seite, Slash-Command-Count ~3.
3
Parallelism
Woche 4+
3–5 git worktrees öffnen, ein Claude in jedem. PostToolUse-Hook für Formatting. Sichere Commands mit /permissions pre-allowen, .claude/settings.json in git eincheckcen.
Graduate when: du frustriert bist, auf einen Claude zu warten.
4
Subagents
Monat 2+
.claude/agents/code-simplifier.md und verify-app.md hinzufügen. "Use 5 subagents to explore the codebase" auf neuen Repos. Adversariales Review-Pattern.
Graduate when: du mehr Zeit mit Switching zwischen Claudes verbringst als die Claudes mit Arbeit.
5
Autonomy
Monat 3+
Auto Mode (Opus 4.7), /focus, Recaps an. /loop für PR-Babysitting, Slack-Feedback. /schedule für Jobs, die geschlossene Laptops überleben. Chrome-Extension für Frontend-Verifikation. /voice.
Endstand: Claude Code ist kein Tool mehr — sondern der Compute-Budget deines Teams.

Umstrittene Empfehlungen

Wo Boris' Setup von seinem eigenen Team abweicht — und wo du selbst entscheiden musst.

Worktrees vs. Checkouts

Boris persönlich nutzt mehrere Checkouts. Der Rest des Teams bevorzugt Worktrees und hat dafür native Worktree-Support gebaut. Beide funktionieren — wähle nach mentalem Modell.

MCP vs. CLI/Skills

Boris nutzt Slack MCP. Sekundäre Quellen aus dem Team pushen zurück: "Für read-heavy Workflows ist ein well-crafted CLI-Skill oft simpler." Faustregel: lokal & simpel → Slash Command. Externe Daten → MCP.

Opus vs. Sonnet

Boris nutzt Opus 4.5/4.6/4.7 mit Thinking für alles. Team-Default und Pro-Tier surface'n weiterhin Sonnet. Faustregel: weniger Steuerungsaufwand bei Opus = oft schneller, trotz höherer Latenz.

Auto Mode vs. Explicit Permissions

Auto Mode (Opus 4.7) auto-approved sichere Permissions. Sicherheitsbewusste Teams bevorzugen manuelles Approve via PermissionRequest-Hook. Sandboxes erlauben Mittelweg.
